=================================================
                        📜 Aurum - 中医病历智能整理工具
=================================================

                            1. 这是什么工具？

你跟诊后，文件夹通常是这样的：

  2025-06-01 曙光医院/
  ├── 张三/
  │   ├── 病历.docx
  │   └── 舌象.jpg
  ├── 李四/
  │   └── 病历.docx
  2025-06-15 曙光医院/
  ├── 张三/
  │   └── 病历.docx
  └── 王五/
      └── ...

想查张三的所有就诊记录，得在十几个日期文件夹里翻来翻去。

Aurum 帮你做三件事：
  1. 把上面的文件夹自动重组为“医院/患者/就诊日期”的结构。
  2. 把所有病历信息存入数据库，支持搜索、筛选、分组。
  3. 用 AI 帮你识别证型、方剂名，自动生成按语。

=================================================
                            2. 拿到软件后怎么用？

  1. 解压整个文件夹到你的电脑上（不要只把 Aurum.exe 拖出来，整个文件夹
     要在一起）。
  2. 双击 Aurum.exe，允许访问公共网络。
  3. 等待几秒钟，浏览器会自动打开，显示 http://localhost:8501。
  4. 看到登录页面，说明启动成功了。

注意：第一次启动会稍慢（10 秒左右），是正常的。后续启动会快一些。

=================================================
                            3. 第一次使用做什么？

  ① 注册账号
     - 点击“注册新账号”，输入用户名和密码（密码至少 6 位）。
     - 注册成功后，用刚注册的账号登录。

  ② 填入你的 DeepSeek API 密钥（可选，但推荐）
     - 登录后，在“🤖 智能体”标签页里，展开“🔑 账号 API 密钥管理”。
     - 输入你在 DeepSeek 平台申请的 API 密钥，点击保存。
     - 不填也能用归档和数据库浏览功能，只是 AI 识别功能用不了。

     【为什么要填 API 密钥？】
     AI 识别（证型、方剂、按语）需要调用 DeepSeek 的大模型，
     这是付费服务，需要你自己充值。本工具不收取任何额外费用。

  ③ 开始归档
     - 在“📂 归档整理”标签页，选择你的源目录（比如 D:/2025年病历）。
     - 选择或生成目标目录（默认会在源目录旁边自动生成一个
       “_Aurum归档” 文件夹）。
     - 点击“执行归档整理”，等待完成。

=================================================
                            4. 两种归档模式的区别

  Aurum 提供了两种方式处理你的病历，根据情况选择：

  【归档整理】（主界面）
  - 适用场景：你手里的是**未整理**的日期文件夹
  - 会做什么：复制文件到目标目录，生成患者档案，建立数据库索引
  - 原始文件不会被移动或删除

  【刷新数据库】（折叠面板，在“归档整理”页面下方）
  - 适用场景：你已经有了**已整理好**的归档结构（医院/患者/日期）
  - 会做什么：只更新数据库中的文件路径和病历内容，不会创建或修改任何文件夹
  - 会同步更新患者档案（_患者档案.txt）
  - 如果你只是新增了一个患者或修改了病历内容，用这个最快

                            5. 文件都放在哪里？

  内容                    存放位置
  --------------------------------------------------------------
  程序本身                你解压后的整个文件夹（所有文件要在一起）
  数据库                  aurum_index.db（和 Aurum.exe 在同一目录）
  配置文件                config.yaml（和 Aurum.exe 在同一目录）
  归档后的文件            你自己指定的“目标目录”
  临时文件                系统临时目录（自动清理，不用管）

  重要：
  数据库和配置文件永远在 Aurum.exe 旁边。如果你想备份数据，直接复制
  aurum_index.db 和 config.yaml 即可。

  如果解压后把 Aurum.exe 单独拖到桌面运行，程序会找不到数据库和配置，
  导致出错。请务必让整个文件夹保持在一起。

                            6. 如何更新到新版本？

  ① 检查更新
     - 登录后，在左侧边栏点击“🔍 检查更新”。
     - 如果有新版本，会弹出提示。

  ② 自动更新（Windows）
     - 点击“立即更新”，程序会自动下载新版本并替换旧文件。
     - 你的数据库（aurum_index.db）和配置文件（config.yaml）不会被覆盖，数据不丢。
     - 更新完成后，程序会自动重启。

  ③ 手动更新（Mac 或自动更新失败时）
     - 前往 GitHub Releases 下载最新版本。
     - 解压后，将新的 Aurum.exe 复制到旧文件夹中，覆盖同名文件。
     - 如果更新了核心模块，可能需要替换整个 aurum_core 文件夹（通常只替换 exe 即可）。

=================================================
                            7. 我不想用了，怎么删？

  直接删除整个文件夹即可。Aurum 不会往系统盘写任何东西，不会留下垃圾文件。

  如果你想保留数据但卸载程序，把 aurum_index.db 和 config.yaml 复制到别处，
  然后删除其他文件即可。

=================================================
                            8. 注意事项（重要！）

  ① 不要把 Aurum.exe 单独拖出来
     整个文件夹要在一起，否则程序会找不到数据库和配置文件。

  ② 移动整个文件夹时
     数据库和配置会跟着走，数据不丢。但不要只移动 exe，要移动整个文件夹。

  ③ API 密钥是付费的
     本工具调用 DeepSeek 的 API 是需要你自己充值的。建议先在 DeepSeek 官网
     充值少量金额试用。

  ④ 数据安全
     Aurum 不会把你的病历上传到任何云端。唯一发到网上的内容是你脱敏后发给
     DeepSeek API 的医学描述文本，且不包含任何个人身份信息。

  ⑤ 杀毒软件误报
     由于打包方式的特性，部分杀毒软件可能会误报。这是正常现象，请将 Aurum
     添加到信任列表。

  ⑥ 端口占用
     如果 8501 端口被占用，程序可能无法启动。可以关闭其他占用该端口的程序，
     或修改 run.py 中的端口号。

  ⑦ 自动更新仅支持 Windows
     Mac 用户请手动下载新版本替换。

                            9. 常见问题

  Q：双击 Aurum.exe 没反应怎么办？
  A：打开任务管理器，看看有没有 Aurum.exe 或 python 进程在运行。如果有，
     可能是浏览器没自动打开，手动输入 http://localhost:8501 访问。

  Q：提示“无法找到 config.yaml”怎么办？
  A：检查一下 config.yaml 是否和 Aurum.exe 在同一个文件夹里。如果是第一次运行，
     可能还没生成，可以重新解压完整的文件夹。

  Q：换了电脑，数据怎么迁移？
  A：把整个 Aurum 文件夹复制到新电脑即可。如果只想迁移数据，复制
     aurum_index.db 和 config.yaml 到新电脑的 Aurum 目录下。

  Q：忘记密码怎么办？
  A：在登录页面点击“忘记密码”，输入用户名并重置密码。

  Q：归档后的文件在哪里？
  A：你在“归档整理”页面指定的“目标目录”就是归档后的位置。如果你没有手动改，
     默认会在源目录旁边生成一个“_Aurum归档”文件夹。

  Q：AI 识别提示“API 密钥未保存”怎么办？
  A：在“智能体”页面展开“API 密钥管理”，填入你的 DeepSeek API 密钥并保存。

  Q：自动更新下载失败怎么办？
  A：网络不稳定可能导致下载失败。可以重新点击“检查更新”重试，或前往 GitHub
     Releases 手动下载并覆盖。

  Q：Mac 电脑能用吗？
  A：可以。但需要另外下载 Mac 版本的压缩包（.app），操作方式与 Windows 类似。

                            10. 下载速度慢怎么办？

  如果你在下载 Aurum 时速度很慢，可以尝试以下方法：

  ① 使用浏览器扩展（推荐）
     安装 GitHub Accelerator 等插件，自动加速下载。

  ② 使用镜像站
     在下载链接前加上以下任意一个前缀：
     - https://ghproxy.com/
     - https://ghproxy.net/
     - https://gitproxy.click/
     例如原链接为 https://github.com/.../Aurum.zip，
     替换为 https://ghproxy.com/https://github.com/.../Aurum.zip 即可。

  ③ 使用下载工具
     如 dlg 等支持代理的工具，可自动选择最快节点。

  如果以上方法仍无法解决，请尝试其他网络环境或联系技术支持。
